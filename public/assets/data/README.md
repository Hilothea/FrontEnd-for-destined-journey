# 数据录入指南

本目录用于录入预设的装备、道具和技能数据，使用 JSON5 格式（支持注释的 JSON），方便不懂代码的人员维护。

## 📁 文件结构

```
public/assets/data/
├── README.md          # 本说明文档
├── equipments.json    # 装备数据
├── items.json         # 道具数据
├── skills.json        # 技能数据
├── destinedOnes.json  # 命定之人数据
└── backgrounds.json   # 初始剧情数据
```

## ✨ JSON5 特性

**本项目使用 JSON5 格式，支持以下特性：**

1. **✅ 支持注释** - 这是最重要的特性！

   ```json5
   {
     // 这是单行注释
     "name": "物品名称",
     /* 这是多行注释
        可以写很多行 */
     "cost": 10
   }
   ```

2. **✅ 允许尾随逗号**

   ```json5
   {
     "name": "物品名称",
     "cost": 10,  // 最后一个属性后可以有逗号
   }
   ```

3. **✅ 更灵活的字符串**
   - 可以使用单引号
   - 支持多行字符串

## 🚀 快速开始

1. **直接编辑数据文件**：
   - 打开 `equipments.json`、`items.json`、`skills.json`、`destinedOnes.json`、`backgrounds.json`
   - 按照格式添加或修改数据
   - **可以自由添加注释来说明数据用途**

2. **提交到仓库**：
   - 将修改提交到 GitHub 仓库的 `public/assets/data/` 目录
   - CDN 会自动同步最新的数据文件（可能需要几分钟）

3. **验证数据**：
   - 刷新页面查看更新后的数据
   - 系统会自动绕过缓存获取最新数据

## ⚠️ 重要提示

- ✅ **可以随意添加注释** - 用 `//` 或 `/* */` 添加说明
- ✅ **尾随逗号无所谓** - 最后一个属性后可以有逗号
- 📝 **什么文件写什么数据**：
  - `equipments.json` - 装备数据
  - `items.json` - 道具数据（货币数据在代码中，如需修改请联系相关技术人员）
  - `skills.json` - 技能数据

## 📝 数据格式说明

## 🎨 品质等级 (rarity)

数据中的 rarity 字段表示品质等级，可选值有：

- `common` - 普通
- `uncommon` - 优秀
- `rare` - 稀有
- `epic` - 史诗
- `legendary` - 传说
- `mythic` - 神话
- `only` - 唯一

**注意：该字段需要填写对应的英文而非中文**

### 装备数据 (equipments.json)

```json5
{
  // 装备按分类组织
  "武器": [
    {
      "name": "长剑",  // 装备名称
      "cost": 10,      // 转生点消耗
      "type": "武器",  // 装备类型
      "rarity": "common",  // 品质等级
      "tag": "[单手剑][攻击: 50][力量+2]", // [子类型][攻击/防御: XXX][属性+1/2]
      "effect": "(具体的、可代入公式或战斗流程的数值化效果)",
      "description": "(装备的叙事性描述)",
    },
    // 可以继续添加更多装备...
  ],
  "防具": [
    // 防具数据...
  ],
}
```

### 道具数据 (items.json)

```json5
{
  // 道具按分类组织
  "消耗品": [
    {
      "name": "治疗药水",  // 道具名称
      "cost": 5,           // 转生点消耗
      "type": "消耗品",    // 道具类型
      "rarity": "uncommon", // 品质等级
      "tag": "[恢复][HP+50]", // [标签]（非必填，若道具为武器则需要填写，参考武器数据）
      "effect": "(道具的具体的数值化功能描述)",
      "description": "(道具的叙事性描述)",
    },
  ],
  // 注意：货币相关数据部分保留在代码中
}
```

### 技能数据 (skills.json)

**支持使用中文或英文字段名**：

- 中文格式：`"主动技能"` 和 `"被动技能"`
- 英文格式：`"ActiveSkills"` 和 `"PassiveSkills"`

```json5
{
  // 支持中文字段名
  "主动技能": {
    "战技": [  // 子分类
      {
        "name": "强力斩击",  // 技能名称
        "cost": 15,          // 转生点消耗
        "type": "主动",      // 技能类型
        "rarity": "rare",    // 品质等级
        "tag": "[力量][单体][物理攻击][威力: 150][破甲]", // [关联属性][目标类型][核心功能][威力: XXX][可选机制]
        "consume": "[攻击: 50SP]",  // 消耗说明
        "effect": "(具体数值化效果。伤害必须明确类型，如：造成150点伤害，70%物理伤害，30%能量伤害)",
        "description": "(叙事性描述)",
      },
    ],
    // 其他子分类：法术、祷告、其它...
  },
  // 也支持英文字段名
  "PassiveSkills": {
    "战技": [
      // 被动技能数据...
    ],
  },
}
```

### 初始剧情数据 (backgrounds.json)

```json5
      "name": "（初始剧情标题）",  // 为初始剧情起个标题好进行区分

    }
```

```json5
}
```

## ⚠️ 注意事项

1. **JSON5 格式提示**：
   - 字符串可以用单引号 `'` 或双引号 `"`
   - 数字不需要引号
   - 最后一个属性后的逗号可有可无
   - 中文标点符号要用英文标点
   - **可以自由添加注释说明数据用途**

2. **装备/道具/技能必填字段**：
   - `name`：名称（不能为空）
   - `cost`：消耗的转生点数（数字）
   - `type`：类型
   - `rarity`：品质
   - `tag`: 标签
   - `effect`：效果
   - `description`：描述

3. **特殊字段**：
   - 技能数据需要额外的 `consume` 字段（消耗说明）

4. **分类名称**：
   - 装备：武器、防具、饰品等
   - 道具：消耗品、材料等
   - 技能：主动技能和被动技能，每个下面还有战技、法术、祷告、其它等子分类（可自定义子分类）

## ❓ 常见问题

**Q: 为什么我的数据没有生效？**
A: 请确保：

1. 文件已成功提交到 GitHub 仓库
2. 等待几分钟让 CDN 更新缓存
3. 刷新页面查看最新数据

**Q: 可以在 JSON 文件里写注释吗？**
A: ✅ 可以！本项目使用 JSON5 格式，完全支持注释。使用 `//` 写单行注释，使用 `/* */` 写多行注释。

**Q: 最后一个属性后面的逗号要不要删除？**
A: 不需要！JSON5 支持尾随逗号，保留或删除都可以。

**Q: 如何添加新的分类？**
A: 在对应文件中按照现有格式添加新的分类名称和数据即可。建议添加注释说明分类用途。

如有其他问题，请联系技术人员。
